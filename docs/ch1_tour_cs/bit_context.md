# 1.1 정보는 비트와 컨텍스트로 이루어진다
---
### 이 장에서 다루는 내용

> 프로그램이 파일 형태로 저장되는 과정과, 텍스트 정보가 비트 단위로 표현되는 방식을 이해한다.

* 프로그램은 결국 비트들의 집합이며, 그 해석은 문맥(context)에 따라 달라진다.  
* 모든 프로그램 데이터가 “비트 + 문맥”으로 구성된다는 개념을 이해하면 파일, 메모리, 네트워크 데이터를 시스템 수준에서 올바르게 해석할 수 있다.

### 주요 키워드

## 1. 소스 프로그램의 탄생

프로그램은 프로그래머가 편집기(editor)를 사용하여 작성한 **소스 프로그램(source program)** 으로부터 시작된다. 소스 코드는 일반적인 텍스트 파일로 저장되며, 예를 들어 `hello.c` 같은 형태를 가진다.

소스 프로그램의 본질은 0과 1의 값을 가지는 **비트(bit)** 들의 연속이며, 이 비트들은 8비트 단위의 **바이트(byte)** 로 조직된다.

## 2. 정보의 표현

대부분의 컴퓨터 시스템은 **ASCII 표준**을 사용하여 문자(character)를 바이트 단위의 정수 값으로 표현한다.  
예를 들어, `#`의 ASCII 값은 35, `i`는 105이다. 따라서 `hello.c` 파일은 이러한 정수 값들의 시퀀스로 저장된다.

| 문자 | ASCII 값 |
| ---- | -------- |
| `#`  | 35       |
| `i`  | 105      |
| `h`  | 104      |

파일의 저장 구조를 비트 수준으로 보면 다음과 같은 형태를 갖는다.
```
01101000 01100101 01101100 01101100 01101111
```

위 시퀀스는 문맥(context)에 따라 문자열 `"hello"`로 해석되며, 단순한 비트 덩어리로만 존재하는 데이터와 구별된다.

## 3. 문맥의 중요성

시스템의 모든 정보는 결국 **비트들의 모음**이다.  
디스크에 저장된 파일, 메모리에 로드된 데이터, 실행 중인 명령어도 모두 비트다.

데이터가 실제로 어떤 의미를 갖는지는 오직 문맥(context)에 의해 결정된다.
- 동일한 바이트 시퀀스가 메모리에 있을 때,
  - 정수형으로 해석되면 `int` 데이터가 되고,
  - 문자로 해석되면 문자열(string)이 되며,
  - 명령어로 해석되면 기계어(machine instruction)가 된다.

즉, **비트 자체는 중립적인 정보 표현이며, 문맥이 의미를 부여한다.**

> 이 개념은 이후 어셈블리, 메모리 구조, 파일 시스템 등 다양한 주제의 기반이 된다.

## 4. 예제 코드

아래는 가장 기본적인 C 프로그램 예시이다.  
이 파일 또한 문자로 이뤄진 비트의 집합이다.
```c
#include <stdio.h>

int main()
{
  printf("hello, world\n");
  return 0;
}
```

`printf` 문에서 문자열 `"hello, world\n"` 은 내부적으로 각 문자의 ASCII 값으로 변환되어 저장된다.
