# #8 예외적인 제어흐름

## 이 장에서 다루는 내용

* **간단한 설명**: 프로그램 실행 도중 발생하는 외부 이벤트나 비정상 상황에 대응하기 위한 *예외적인 제어 흐름*의 개념을 다룬다.  
* **학습 이유**: 하드웨어와 운영체제가 협력하여 제공하는 제어 흐름 전환 방식을 이해함으로써, 시스템 수준의 동작 원리를 명확히 알 수 있고 안정적인 프로그램을 작성할 수 있다.

## 1. 예외 (Exceptions) (8.1)

프로세서가 실행 흐름을 중단하고 **예외 처리 루틴**으로 분기하는 사건.  
- **동기적 예외**: 실행 중인 명령어에서 직접 발생 (0으로 나누기, 페이지 폴트)  
- **비동기적 예외**: 외부 장치에서 발생 (타이머, I/O 인터럽트)  
- 종류: Interrupt, Trap, Fault, Abort  

---

## 2. 시그널 (Signals) (8.2)

운영체제가 프로세스에게 전달하는 **비동기 알림 메커니즘**.  
- 기본 동작: 종료, 무시, 중단  
- 사용자 정의 핸들러를 등록하여 특정 이벤트 처리 가능  
- 예: `SIGINT`, `SIGKILL`, `SIGCHLD`  

---

## 3. 시스템 콜 (System Calls) (8.3)

사용자 프로그램이 운영체제 서비스를 요청할 때 사용하는 인터페이스.  
- **트랩(trap)**을 통해 커널 모드로 진입  
- 파일 I/O, 프로세스 생성(`fork`), 실행(`execve`) 등 다양한 기능 제공  

---

## 4. 컨텍스트 스위치 (Context Switch) (8.4)

멀티태스킹 운영체제에서 CPU가 다른 프로세스로 제어권을 전환하는 과정.  
- 타이머 인터럽트 발생 → 커널介입 → 현재 프로세스 상태 저장 → 다른 프로세스 상태 복원  
- 이를 통해 여러 프로세스가 CPU를 **시분할(Time Sharing)** 방식으로 공유 가능  

---

8장은 **예외적인 제어 흐름**이 어떻게 시스템 반응성과 안정성을 보장하는지 보여준다.  
이는 단순한 프로그램 실행 모델을 넘어, **운영체제·하드웨어·응용 프로그램**이 긴밀히 협력하는 방식을 이해하는 데 핵심적인 내용이다.
