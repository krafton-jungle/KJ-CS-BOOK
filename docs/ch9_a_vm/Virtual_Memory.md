# 가상 메모리(Virtual Memory)

# 개념
	가상 메모리는 실제 주기억장치(RAM)보다 더 큰 메모리 공간을 추상적으로 제공하는 기술이다.
	프로그램은 직접 물리 주소를 사용하는 것이 아니라, 운영체제가 제공하는 가상 주소를 사용한다.
	가상 주소는 메모리 관리 장치(MMU)에 의해 물리 주소로 변환되어 실제 메모리에 접근하게 된다.

# 가상메모리를 사용하는 이유
	물리 메모리 용량을 초과하는 프로그램도 실행할 수 있도록 한다.
	여러 프로세스가 동시에 실행될 때, 각 프로세스는 자신만의 독립적인 주소 공간을 가지는 것처럼 보이게 한다.
	이를 통해 프로세스 간 메모리 침범을 방지하고, 보안과 안정성을 높일 수 있다.

# 페이징(Paging)
	가상 메모리와 물리 메모리를 일정한 크기의 블록으로 나누어 관리하는 방식이다.
	가상 메모리의 블록은 페이지(Page), 물리 메모리의 블록은 프레임(Frame)이라 한다.
	MMU는 페이지 테이블을 참조하여 가상 주소의 페이지 번호를 물리 메모리의 프레임 번호로 변환한다.

# 페이지 폴트(Page Fault)
	실행 중인 프로그램이 접근하려는 페이지가 물리 메모리에 존재하지 않을 경우 발생한다.
	이때 운영체제는 보조기억장치에 있는 해당 페이지를 물리 메모리로 적재하고, 페이지 테이블을 갱신한다.
	이러한 과정으로 인해 가상 메모리는 주기억장치와 보조기억장치를 함께 활용하는 구조가 된다.

# 장단점
	장점: 대형 프로그램 실행 가능, 메모리 보호 및 프로세스 간 독립성 보장, 메모리 활용 효율성 향상.
	단점: 디스크 접근이 발생하는 경우 성능 저하가 발생할 수 있다.
